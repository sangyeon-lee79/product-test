# Blueprint: 방울아놀자 (Bang-ul Play)

## 1. 프로젝트 개요

'방울아놀자'는 반려동물 보호자, 서비스 제공자, 그리고 관리자를 위한 종합 SNS 플랫폼입니다. 보호자는 반려동물의 건강, 활동, 이벤트를 타임라인에 기록하고 공유할 수 있습니다. 서비스 제공자는 자신의 서비스를 홍보하고 예약을 관리하며, 보호자와 소통할 수 있습니다. 관리자는 플랫폼의 핵심 데이터를 관리하고 시스템 전반을 운영합니다.

## 2. 현재 구현된 기능 (As-Is)

### V0.3 (Phase A: Admin 기능 고도화)
-   **역할 기반 뷰:** 사용자(User), 공급자(Provider), 관리자(Admin) 역할에 따른 뷰 전환
-   **다국어 지원 (i18n):**
    -   13개국어(ko, en, ja, zh_cn, zh_tw, es, fr, de, pt, vi, th, id_lang, ar) 지원
    -   **자동 번역 UI:** 한국어 기준 12개국어 자동 번역 기능 (GPT/Claude API 연동 가능 구조)
    -   i18n 리소스 CRUD 및 검색 기능
-   **마스터 데이터 관리 (DB 연동):**
    -   **카테고리 관리:** industry, breed, disease, symptom, metric 등 10종 기본 카테고리 지원
    -   **아이템 관리:**
        -   아이템 추가, 수정, 삭제(실제 삭제 및 사용 중일 경우 비활성화)
        -   **계층 구조 지원:** `parent_id`를 통한 아이템 간 부모-자식 관계 설정 (예: 질병-증상-수치 계층화 가능)
        -   13개국어 표시명 일괄 등록 및 자동 번역 연동
-   **국가 및 통화 관리:**
    -   국가별 코드, 표시명, 상태 관리
    -   통화별 코드, 심볼, 소수점 자리수 관리
    -   국가-통화 매핑 (기본 통화 설정 지원)
-   **백엔드 아키텍처:**
    -   Cloudflare Workers + D1 (SQLite) 기반 서버리스 구조
    -   JWT 인증 및 역할 기반 권한 제어 (Admin/Provider/Guardian)
    -   RESTful API 디자인

## 3. 최근 변경 사항 (Simplified Disease System)

-   **질병 관리 체계 단순화:**
    -   기존의 복잡한 4단계 매핑 구조(Disease -> Symptom -> Metric -> Unit/LogType)를 폐지.
    -   별도의 '질병 연결 매핑' 메뉴를 제거하고, 일반 '마스터 데이터' 메뉴에서 통합 관리하도록 변경.
    -   `master_items`의 `parent_id`를 활용하여 필요한 경우에만 자유로운 계층 구조를 형성할 수 있도록 유연성 확보.
-   **아이템 삭제 로직 개선:**
    -   사용자가 아이템을 직접 삭제할 수 있도록 변경.
    -   다른 데이터(예: 반려동물 질병 기록, 로그 등)에서 참조 중인 경우 외래 키 제약 조건에 의해 삭제가 실패하면 자동으로 '비활성화' 상태로 전환하여 데이터 정합성 유지.
