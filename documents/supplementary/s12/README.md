# S12 보충자료 - OAuth 전환 + 마무리

## 1. OAuth 공급자별 설정 체크리스트
- **Google**: 
  - [ ] Google Cloud Console 프로젝트 생성.
  - [ ] iOS/Android/Web Client ID 발급.
  - [ ] 서버용 `id_token` 검증 라이브러리 설정.
- **Apple**:
  - [ ] Apple Developer Program 등록.
  - [ ] App ID 및 Service ID 설정.
  - [ ] Private Key(.p8) 및 Key ID 발급.

---

## 2. 계정 병합 전략 (테스트 -> OAuth)
- **병합 가능**: 테스트 계정의 `email`과 OAuth 계정의 `email`이 일치할 경우 기존 `user_id`에 `oauth_provider`/`oauth_id`를 업데이트한다.
- **병합 불가**: 이메일이 다를 경우 신규 사용자로 가입하며, 기존 데이터 수동 병합 기능은 MVP 범위 외로 둔다.

---

## 3. i18n 누락 점검 및 하드코딩 제거
- **점검 스크립트**: 클라이언트 소스 내 `t('...')` 키를 추출하여 `i18n_translations` 테이블과 비교한다.
- **하드코딩**: UI 텍스트가 `t()` 함수 없이 문자열로 작성된 모든 곳을 찾아 수정한다.
- **최종 검수**: 모든 언어 선택 시 UI가 깨지지 않는지(Overflow 등) 13개국어 전수 검사.

---

## 4. 전체 E2E 시나리오
1. **OAuth 로그인**: Google/Apple 버튼 클릭 후 정상 가입/로그인.
2. **프로필/펫 등록**: 핸들 선점 및 펫 2마리(당뇨/정상) 등록.
3. **질병 기록**: 오프라인 상태에서 혈당 입력 -> 온라인 복귀 시 동기화 확인.
4. **위험 경고**: 저혈당 수치 입력 시 즉시 경고 배너 노출.
5. **예약/바이럴**: Provider 예약 수락 -> 완료 -> 1-click 피드 공유 -> 매장 링크 유입 확인.
6. **광고 정책**: 피드 목록 배너 확인, 혈당 기록 화면 광고 차단 확인.
