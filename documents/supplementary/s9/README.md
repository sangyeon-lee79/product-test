# S9 보충자료 - Provider 매장 + 서비스

## 1. 매장/서비스 입력 스키마 (다국어 포함)
매장 및 서비스 이름/설명은 사용자가 직접 입력한 원본 외에도 다국어 자동/수동 번역을 지원한다.
- **원본 필드**: `name`, `description` (주로 현지어).
- **다국어 필드**: `name_translations`, `description_translations` (JSONB).
  - 예: `{"en": "Happy Grooming", "ko": "해피 미용실"}`.

---

## 2. 업종 (Industry) 다중 선택 정책
- **기본 업종**: 매장의 성격을 결정하는 주 업종 1개 필수.
- **추가 업종**: 병원이면서 미용도 하는 경우 등 `store_industries` 테이블을 통해 다중 연결 가능.
- **검색**: 어떤 업종으로 검색하든 해당 매장이 결과에 포함된다.

---

## 3. 가격 및 통화 표기 규칙
- **통화**: 매장 소속 국가(`country_id`)에 따라 기본 통화(`currency_id`)가 자동 설정된다.
- **표기**: `currencies` 테이블의 `symbol`과 `decimal_places` 정보를 활용하여 UI에서 렌더링한다. (예: ₩ 50,000, $ 45.00)

---

## 4. 할인 정책 (기간 및 우선순위)
- **할인 등록**: 특정 서비스에 대해 할인율(`discount_rate`)과 시작/종료일을 설정한다.
- **우선순위**: 기간이 겹치는 여러 할인이 있을 경우 가장 높은 할인율을 우선 적용한다.
- **노출**: 정가와 할인가를 동시에 표시하여 마케팅 효과를 극대화한다.
