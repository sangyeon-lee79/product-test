# PRD.md - Product Requirements (Bang-ul Play)

## 1. Core Principles
- **Total Admin Control:** 관리자는 플랫폼의 모든 선택값, 번역값, 국가 설정을 데이터베이스 수준에서 통제함.
- **Source of Truth - Master:** 코드 내 하드코딩된 선택값(업종, 견종 등)을 금지하며 모든 값은 `master_items`를 참조함.
- **Source of Truth - Translation:** 모든 표시 텍스트는 `translations` 테이블에서만 기원하며, 실시간 편집 및 배포가 가능해야 함.

## 2. Management Modules (MVP Scope)

### 2.1 마스터 데이터 관리 (Master Data Management)
- **목적:** 업종, 견종, 질병 등 정형 데이터 카테고리와 아이템 관리.
- **UX 패턴:** 카테고리 리스트 -> 카테고리 선택 -> 아이템 리스트 -> 아이템 수정/추가.
- **규칙:** 카테고리 키와 아이템 키는 시스템 내부용이며, 실제 표시명은 번역 시스템과 연동됨.

### 2.2 언어 및 번역 관리 (Language Management)
- **목적:** 플랫폼 전체 텍스트(UI, 시스템 메시지, 마스터명)의 13개 국어 사전 관리.
- **필수 기능:** 키/텍스트 검색, 유형별 필터, 테이블 형태의 다국어 동시 편집.
- **규칙:** 하드코딩된 문자열 발견 시 즉시 제거하고 키 기반 조용으로 전환.

### 2.3 국가 및 통화 관리 (Country Management)
- **목적:** 국가별 기본 통화 설정 및 결제 기준 수립.
- **공급자 규칙:** 공급자는 국가를 선택하면 통화가 자동으로 결정됨 (수정 불가).
- **소비자 규칙:** 모든 상거래 행위(서비스/상품)는 해당 매장의 통화로 기록 및 표시됨.

## 3. UI/UX Rules
- 모든 관리자 화면은 **"리스트업 -> 상세 선택 -> 수정/저장"**의 3단계 흐름을 유지함.
- Material 3 가이드에 따라 8dp 배수 간격과 카드형 UI를 사용함.
