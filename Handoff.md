# 프로젝트 인수인계서 (Handoff.md)

## 1. 프로젝트 개요
- **프로젝트 명**: Pet Life (종합 반려동물 생애주기 관리 플랫폼)
- **핵심 목표**: 반려동물의 건강 기록, 마스터 데이터 관리, 보호자 및 서비스 제공자 간의 연동을 지원하는 다국어(13개국어) 기반 글로벌 서비스.
- **주요 기술 스택**:
  - **Frontend**: React (TypeScript), Vanilla CSS, Web Components.
  - **Backend**: Cloudflare Workers, Hono, D1 (SQLite), Wrangler.
  - **i18n**: DB 기반 동적 번역 시스템 (13개국어 지원).

## 2. 현재 개발 상태 (2026-03-07 기준)
### 완료된 기능 (S1 ~ S7 일부)
- **Admin Web (마스터 데이터 관리)**:
  - 6단계 계층 구조(L1~L6) 마스터 데이터 관리 시스템.
  - 다국어 번역(i18n) 통합 관리 및 자동 번역 연동.
  - 번역 누락 방지 가드(Validation) 및 배포 게이트 적용.
- **Guardian Web (보호자용)**:
  - 펫 프로필 관리 (5단계 위저드 형태의 상세 수정 플로우).
  - 건강 측정 로그(혈당 등) 기록 및 관리 (2단계 위저드 UI).
  - 타임라인 기반의 활동 내역 표시.
- **Backend / DB**:
  - D1 기반 마이그레이션 체계 (0047_...까지 적용 완료).
  - 마스터 데이터 및 i18n 정규화 완료.
  - 외래 키 제약 조건을 고려한 안전한 데이터 클린업 및 시드 로직.

### 진행 중 / 예정 사항 (S8 ~ S12)
- **Provider Web**: 서비스 제공자(병원, 미용 등)용 대시보드 및 예약 관리.
- **Booking 시스템**: 보호자와 제공자 간의 예약 프로세스 구현.
- **Mobile App**: Flutter 기반 모바일 인터페이스 구현 (현재 기초 Scaffold 상태).
- **통계/분석**: 측정 데이터를 기반으로 한 건강 차트 및 리포트 기능.

## 3. 핵심 아키텍처 및 규칙
### i18n (다국어) 원칙
- **하드코딩 금지**: 모든 UI 텍스트는 `i18n_translations` 테이블의 키(`admin.*`, `master.*`, `guardian.*`)를 참조해야 함.
- **한국어 원천(Source)**: 새로운 항목 생성 시 한국어(`ko`) 입력 후 자동 번역 기능을 통해 13개국어를 채우는 방식 권장.
- **UI 렌더링**: 프론트엔드에서는 항상 `t(key)` 함수를 통해 번역된 라벨만 표시하며, 키값이 직접 노출되지 않도록 처리됨.

### 마스터 데이터 구조
- `master_categories` (대분류) - `master_items` (항목들) 계층 구조.
- `parent_item_id`를 통한 트리 구조 지원 (특히 질병군/질병상세 등).
- 모든 마스터 아이템은 `code`를 기준으로 로직에서 참조하며, 표시명은 `i18n` 테이블에서 가져옴.

### 데이터 무결성
- D1 마이그레이션 시 외래 키 제약 조건(`FOREIGN KEY`)이 엄격히 적용되므로, 데이터 삭제/수정 시 참조 순서(Bottom-up)를 준수해야 함.

## 4. 주요 파일 및 디렉토리
- `services/api/src/db/migrations/`: DB 스키마 및 시드 데이터 이력.
- `apps/admin-web/`: 관리자 페이지 (마스터 데이터 및 번역 관리).
- `apps/guardian-web/`: 보호자용 웹 서비스.
- `blueprint.md`: 전체 프로젝트의 기능 설계 및 플랜 문서.
- `GEMINI.md`: AI 개발 가이드 및 환경 설정.

## 5. 실행 및 배포 방법
- **API 서버**: `cd services/api && npm run dev`
- **Frontend**: `cd apps/admin-web && npm run dev`
- **배포**: `wrangler deploy`를 통해 Cloudflare 환경으로 배포.

## 6. 다음 작업자를 위한 조언
- **번역 키 확인**: 새로운 UI 추가 시 반드시 `0042`번 이후의 마이그레이션 패턴을 참고하여 번역 키를 먼저 등록하십시오.
- **위저드 UI**: 펫 수정 및 건강 기록은 복잡한 의존성 필드가 많으므로, 현재 구현된 `StepWizard` 패턴을 유지하는 것이 유지보수에 유리합니다.
- **데이터 보정**: `verify_missing_master_translations.sql` 쿼리를 주기적으로 실행하여 번역이 누락된 마스터 데이터를 점검하십시오.
